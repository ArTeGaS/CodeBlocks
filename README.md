# CodeBlocks

Публічна статична версія гри **“Код-робот у підземеллі”** для GitHub Pages.

Гра українською для дітей приблизно 7 років: дитина складає алгоритм із Blockly-блоків, запускає його, і робот проходить поле з клітинками. Кампанія має 23 послідовні рівні: від одного кроку до умов, циклів, ящиків, кнопок, патрулів і великих фінальних мап.

## Публічна адреса

Після увімкнення GitHub Pages сайт відкривається тут:

```text
https://artegas.github.io/CodeBlocks/
```

Це повністю статичний сайт: без бекенду, акаунтів, бази даних, CDN і зовнішніх картинок.

## Локальний запуск

Двічі клікнути `start_local.bat` або виконати в цій папці:

```bat
python -m http.server 8787
```

Потім відкрити:

```text
http://localhost:8787
```

## Для учнів

Публічна Pages-версія відкривається в учнівському режимі. Панель викладача не займає місце й не додається до інтерфейсу без окремого локального доступу.

## Як грати

1. Обери рівень.
2. Перетягни блоки в поле `Алгоритм`.
3. Приєднай блоки до `коли натиснуто Запуск`.
4. Натисни `Запуск`, `Крок` або `Пауза`.

Рух `рухатися()` завжди робить один крок і штовхає ящик, якщо за ним є місце. Довгі відрізки треба збирати через `повторити`, а рішення з розгалуженням - через `якщо` або `якщо / інакше`.

На частині рівнів є видимий ліміт блоків: маршрут можна перевіряти вільно, але для зарахування алгоритм треба скласти компактно. Прогрес і код кожного рівня зберігаються в браузері після закриття вкладки або перезапуску пристрою.

Рух робота, ящиків і патрулів анімований через легкі CSS-transform переходи. На слабких пристроях або за системного `prefers-reduced-motion` гра автоматично вимикає фонові анімації; ручні режими доступні через `?motion=low` та `?motion=full`.

## Блоки

- `рухатися()` - один крок у напрямку робота.
- `повернути(ліворуч/праворуч)` - змінює напрям.
- `чекати()` - пропускає хід робота, але рухає патруль.
- `взяти ключ()` - бере ключ на клітинці.
- `відкрити двері()` - відкриває двері, якщо ключ уже є.
- `повторити` - повторює вкладені блоки.
- `повторювати до кристала` - виконує вкладені блоки, доки місія не завершена.
- `якщо` - виконує вкладені блоки, коли умова правдива.
- `якщо / інакше` - обирає одну з двох гілок.

Умови перевіряють шлях попереду, ліворуч або праворуч, а також `безодню`, `ворога`, `пастку`, двері чи ключ.

## Рівні

- 1-7: рух, повороти, ключ, двері, повторення й перші умови.
- 8-14: `якщо / інакше`, ящики, кнопки, ворота, активований міст, патруль, енергія та цикл до кристала.
- 15-17: комбіновані малі рівні з ворогом, пасткою, водою й мостом.
- 18-19: перехідні мапи `20x20` з вантажним мостом і патрульним каналом.
- 20-23: повні мапи `30x30` з водою, лавою, лабіринтом і шістьма кристалами по порядку.

Рівні відкриваються послідовно: наступний рівень доступний після проходження попереднього.

## Як замінити графіку

Основні ассети лежать у `assets/svg/`.

Щоб замінити картинку:

1. Підготуй новий SVG або PNG.
2. Назви його так само, як існуючий файл.
3. Замінити файл у `assets/svg/`.
4. Перезавантаж сторінку.

Шляхи до графіки зібрані в `assetMap` у `script.js`.

## Склад папки

- `index.html` - сторінка гри.
- `styles.css` - оформлення.
- `script.js` - рівні й логіка.
- `vendor/blockly/` - локальні файли Blockly.
- `assets/svg/` - замінні SVG-ассети.
- `.nojekyll` - щоб GitHub Pages не чіпав статичні файли Jekyll-обробкою.
