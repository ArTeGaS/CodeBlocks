# CodeBlocks

Публічна статична версія гри **“Код-робот у підземеллі”** для GitHub Pages.

Гра українською для дітей приблизно 7 років: дитина складає алгоритм із Blockly-блоків, запускає його, і робот проходить поле з клітинками. Перші 10 рівнів ознайомлювальні, рівні 11-14 додають повноцінні ігрові задачі з ворогами, водою, лавою, пастками, лабіринтом і кристалами в правильному порядку.

## Публічна адреса

Після увімкнення GitHub Pages сайт відкривається тут:

```text
https://artegas.github.io/CodeBlocks/
```

Це повністю статичний сайт: без бекенду, акаунтів, бази даних, CDN і зовнішніх картинок.

## Локальний запуск

Двічі клікнути `start_local.bat` або виконати в цій папці:

```bat
python -m http.server 8787
```

Потім відкрити:

```text
http://localhost:8787
```

## Для викладача

Панель `Для викладача` у цій Pages-версії увімкнена за замовчуванням. Вона дає:

- вставлення готового розв'язку рівня;
- скидання прогресу;
- повне очищення локальних даних на цьому браузері.

Якщо треба показати учнівський вигляд без викладацької панелі, відкрий:

```text
https://artegas.github.io/CodeBlocks/#student
```

Повернути викладацький вигляд:

```text
https://artegas.github.io/CodeBlocks/#teacher
```

## Як грати

1. Обери рівень.
2. Перетягни блоки в поле `Алгоритм`.
3. З'єднай блоки як у Scratch.
4. Натисни `Запуск`, `Крок` або `Пауза`.

Рух `рухатися()` завжди робить один крок. Довгі відрізки треба збирати через `повторити`, а рішення з розгалуженням - через `якщо`.

## Блоки

- `рухатися()` - один крок у напрямку робота.
- `повернути(ліворуч/праворуч)` - змінює напрям.
- `взяти ключ()` - бере ключ на клітинці.
- `відкрити двері()` - відкриває двері, якщо ключ уже є.
- `повторити` - повторює вкладені блоки.
- `якщо` - виконує вкладені блоки, коли умова правдива.

Умови перевіряють, що попереду: `шлях`, `безодня`, `ворог`, `пастка`, двері або ключ.

## Рівні

- 1-10: ознайомлення з рухом, поворотами, ключем, дверима, повтореннями й умовами.
- 11: вороги, вода, мости.
- 12: пастки, лава, мости.
- 13: складний лабіринт.
- 14: кристали з номерами, які треба зібрати в правильному порядку.

Рівні відкриваються послідовно: наступний рівень доступний після проходження попереднього.

## Як замінити графіку

Основні ассети лежать у `assets/svg/`.

Щоб замінити картинку:

1. Підготуй новий SVG або PNG.
2. Назви його так само, як існуючий файл.
3. Замінити файл у `assets/svg/`.
4. Перезавантаж сторінку.

Шляхи до графіки зібрані в `assetMap` у `script.js`.

## План заняття на 3 години

- 0:00-0:15 - ідея алгоритму: робот робить тільки те, що йому сказали.
- 0:15-0:45 - рівні 1-2: рух і повороти.
- 0:45-1:15 - рівні 3-4: стіни, ключ, двері.
- 1:15-1:25 - перерва.
- 1:25-2:00 - повторення.
- 2:00-2:35 - умови `якщо` і помилки в алгоритмі.
- 2:35-2:55 - складніші рівні парами.
- 2:55-3:00 - короткий підсумок.

## Типові проблеми

- Сторінка відкрилась, але ассети не видно: запускай локально через HTTP-сервер, не подвійним кліком по HTML.
- Учень зламав алгоритм: натиснути `Очистити` або вставити розв'язок у панелі `Для викладача`.
- Рівень недоступний: треба пройти попередні рівні або скинути прогрес у панелі викладача.

## Склад папки

- `index.html` - сторінка гри.
- `styles.css` - оформлення.
- `script.js` - рівні й логіка.
- `vendor/blockly/` - локальні файли Blockly.
- `assets/svg/` - замінні SVG-ассети.
- `.nojekyll` - щоб GitHub Pages не чіпав статичні файли Jekyll-обробкою.
